1- File System 

    Blocking vs non-blocking code >>>

    1- blocking => when javascripting operation waits until a non-js operation completes
        => whole thing stops
        => when seeing "Sync" name its probably a blocking code
        => example ==> readFileSync

    2- non-blocking => let the code above and beneath executed then iam executed


// Practical
// example, read file 

// method 1 => (sync)
const content = fs.readFileSync('D:\\data\\node\\node\\file.txt', 'utf8');
console.log("Content =>", content);

// method 2 => (Async)
fs.readFile('node\\file.txt', 'utf8', (err, data) =>{

    if (err)
        console.log("Error reading file", err);
    else 
        console.log("Data =>", data);

})
